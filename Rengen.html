Event Manager

The event system is built around a lightweight circular queue to avoid costly memory operations such as reallocations or data shifting. Events are pushed into the queue and persist for a full update cycle, remaining available until they reach the front of the buffer, where they survive for an additional frame. This design ensures that all interested game objects have sufficient time to process incoming events, regardless of update order. By keeping the system simple and cache-friendly, it minimizes overhead while maintaining reliable event delivery.


//HEADER

    class EventManager : public Singleton
    {
        static constexpr int MAX_EVENTS = 16;
    
        int m_Head{ 0 };
        int m_Tail{ 0 };
    
        std::array, MAX_EVENTS> m_Events{};
    
    public:
    
        // Preferred: pass ownership in
        void AddEvent(std::unique_ptr e);
        Event* GetEvent() const;
        void ProcessEvents();
        bool HasEvent() const { return m_Head != m_Tail; }
    };
  

//CPP FILE
        
    void dae::EventManager::AddEvent(std::unique_ptr e)
    {
        assert(e);
        
        const int nextTail = (m_Tail + 1) % MAX_EVENTS;
        
        assert(nextTail != m_Head);
        
        m_Events[m_Tail] = std::move(e);
        m_Tail = nextTail;
    }
        
    Event* dae::EventManager::GetEvent() const
    {
        if(m_Head == m_Tail) return nullptr;
        
        return m_Events[m_Head].get();
    }
        
        
    void dae::EventManager::ProcessEvents()
    {
        if (m_Head == m_Tail) return;
        
        m_Events[m_Head].reset();
        m_Head = (m_Head + 1) % MAX_EVENTS;
    }

Game Objects

Game objects are fundamental to any game engine, so their implementation must be both performant and dependable. To minimize overhead, I use a dirty flag system for transform updates, allowing global positions to be recalculated only when changes occur, rather than every frame. This system also supports hierarchical transforms, where each object’s world position is derived from its local position combined with its parent’s transform. When a parent object moves, its children are marked dirty, ensuring their global positions are updated only when accessed. By combining lazy evaluation with parent-child propagation, this approach avoids redundant calculations and scales efficiently even in complex scene hierarchies.



glm::vec3 dae::GameObject::GetGlobalPos()
{
    if (m_IsMarkedForDeletion)
    return{ 0,0,0 };

    if (m_GlobalPosFlag)
    {
        glm::vec3 newPos{m_LocalPosition};
        std::shared_ptr current = this->m_Parent;

        while (current)
        {
            if (!current->m_GlobalPosFlag)
            {
                newPos += current->m_GlobalPosition;
                current = nullptr;
            }
            else
            {
                newPos += current->m_LocalPosition;
                current = current->m_Parent;
            }
        }

        m_GlobalPosition = newPos;
        m_GlobalPosFlag = false;
    }

    return m_GlobalPosition;
}
